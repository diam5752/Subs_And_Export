## 2024-01-01 - Infinite FFmpeg Hang Prevention
**Vulnerability:** The application executed external FFmpeg processes for audio extraction and video encoding without explicit hard timeouts. While cancellation checks existed, a process that hung indefinitely (e.g. due to corrupted input or deadlocks) would consume resources forever if not manually cancelled.
**Learning:** `subprocess.Popen` in Python does not enforce timeouts by default. Polling loops checking for cancellation or output are insufficient if the process itself hangs and stops producing output or responding to signals.
**Prevention:** Always wrap `subprocess` calls in logic that checks `time.monotonic()` against a maximum allowed duration, and explicitly `kill()` the process if it exceeds this limit. Added `timeout` parameters to `extract_audio` and `run_ffmpeg_with_subs` defaulting to safe upper bounds (5m and 60m respectively).
