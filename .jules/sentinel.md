## 2024-04-18 - [FFmpeg Hang Timeout]
**Vulnerability:** Long-running FFmpeg subprocesses (`run_ffmpeg_with_subs` and `extract_audio`) lacked explicit timeouts. While they had cancellation checks, if the subprocess itself hung (e.g. deadlock, corrupted file causing loop) without user cancellation, the worker thread would be tied up indefinitely. This could lead to resource exhaustion (DoS).
**Learning:** `subprocess.run` has a `timeout` argument, but `subprocess.Popen` (used for streaming output) does not. It requires manual timeout management in the polling loop using `time.monotonic()`.
**Prevention:** Always implement a "hard timeout" check in `subprocess.Popen` polling loops. Calculate reasonable timeouts based on media duration (e.g., 20x duration for encoding) with a generous minimum (e.g., 30-60 mins) to prevent infinite hangs while allowing legitimate long jobs.
