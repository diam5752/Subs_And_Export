## 2025-12-29 - Path Traversal in `ensure_job_size`
**Vulnerability:** The `ensure_job_size` function in `job_routes.py`, `export_routes.py`, and `videos.py` allowed path traversal. It trusted `job.result_data["video_path"]` and cleaned it by lstripping `/` and `static/` or `data/` prefixes, but did not prevent `..` segments. This allowed an attacker to access file sizes of any file on the system readable by the backend process.
**Learning:** Even internal data fields like `result_data` (stored in the database) should be treated as untrusted if they are used to access the file system, especially if they can be influenced by users or complex processing logic.
**Prevention:** Always use `path.resolve()` and `path.is_relative_to(base_dir)` when resolving paths from partial inputs. Never rely on string manipulation alone for path security.
